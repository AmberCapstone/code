# STM32U0 - PlatformIO Integration

The U0 series is not natively supported by PlatformIO so we do some manual configuration.

## STM32CubeMX Project Settings

### Project

- Application Structure: Basic
- Do not generate main(): Unchecked
- Toolchain / IDE: Makefile

### Code Generator

STM32Cube MCU packages and embedded software packs: Copy only the necessary library files

Generated Files:
- CHECK Generate peripheral initializations as a pari of `.c/.h` files per peripheral
- UNCHECK Backup prevously generated files when re-generating
- CHECK Keep User Code when re-generating
- CHECK Delete previously generated files when not re-generated

## STM32CubeMX Generation

PlatformIO doesn't include the U0 Drivers. They will be generated by CubeMX but are ~120k lines long so we don't track them with Git. After the first "Generate Code" you only need to regenerate after changing the IOC.

Run `. post_cubemx.sh` after each "Generate Code."

## PlatformIO Integration

Symlink the `code/lib/platformio/boards` into the same directory as `platformio.ini`. This file describes some compilation, upload, and debug flags for the U0 boards.

See the `platformio.ini` file in this directory for the config options.
